## 📖 簡介

易潔寶管理後台是一個**本地運行的內容管理系統**，讓非技術人員也能輕鬆管理網站內容。

### ✨ 主要特點

- 🎨 **類似 WordPress** 的使用體驗
- 💾 **本地儲存** - 所有資料存在您的電腦上
- 📦 **離線使用** - 不需要連接網路
- 🔒 **安全** - 沒有外部資料庫，完全掌控
- 📤 **匯出/匯入** - 方便備份與還原

### 🔄 工作流程

```
1. 啟動管理後台（本地）
   ↓
2. 編輯內容（類似 WordPress）
   ↓
3. 匯出資料備份
   ↓
4. 建置網站
   ↓
5. 發布到網路
```

## 🚀 如何進入管理後台

### 步驟 1：啟動開發伺服器

打開「命令提示字元」(Windows) 或「終端機」(Mac/Linux)，執行：

```bash
# 進入專案目錄
cd 178mat

# 啟動管理後台
npm run dev
```

### 步驟 2：開啟瀏覽器

開發伺服器啟動後，會顯示：

```
➜  Local:   http://localhost:3000/178mat/
```

### 步驟 3：訪問管理後台

在瀏覽器中輸入：

```
http://localhost:3000/178mat/admin
```

就會看到管理後台的登入畫面！

## 📁 工程實績管理

### 查看所有工程

1. 點擊左側導航的「工程實績」
2. 可以看到所有工程案例的列表
3. 每個工程顯示：
   - 縮圖
   - 標題
   - 地點
   - 年份
   - 描述
   - 標籤

### 搜尋工程

使用頂部的搜尋框，可以搜尋：
- 工程名稱
- 地點
- 城市

### 刪除工程

1. 找到要刪除的工程
2. 點擊「刪除」按鈕
3. 確認刪除

⚠️ **注意**：刪除後無法復原，建議先匯出備份！

### 編輯工程（目前版本）

目前版本的編輯功能正在開發中，暫時需要透過以下方式編輯：

1. 點擊頂部的「匯出資料」按鈕
2. 下載 `cms-data.json` 檔案
3. 使用文字編輯器（如 VSCode、記事本）編輯 JSON 檔案
4. 編輯完成後，點擊「匯入資料」上傳修改後的檔案

#### JSON 檔案結構範例

```json
{
  "projects": [
    {
      "id": "unique-id",
      "title": "工程名稱",
      "location": "完整地址",
      "city": "城市",
      "district": "區域",
      "year": 2024,
      "images": [
        "/images/project-1.jpg"
      ],
      "description": "工程描述...",
      "tags": {
        "buildingType": ["住宅大廈"],
        "floorMaterial": ["地磚"],
        ...
      }
    }
  ]
}
```

## 💾 資料備份與匯出

### 為什麼要備份？

- ✅ 防止意外刪除
- ✅ 方便還原舊版本
- ✅ 轉移到其他電腦
- ✅ 版本控制

### 如何匯出資料

1. 點擊頂部工具列的「匯出資料」按鈕
2. 瀏覽器會自動下載 `cms-data.json` 檔案
3. 將檔案存到安全的地方（建議定期備份）

### 如何匯入資料

1. 點擊頂部工具列的「匯入資料」按鈕
2. 選擇之前匯出的 `cms-data.json` 檔案
3. 系統會自動載入資料
4. 頁面會重新整理，顯示匯入的內容

### 重置資料

如果資料有問題，可以點擊「重置」按鈕：
- ⚠️ 這會清除所有修改
- ⚠️ 還原到原始預設資料
- ⚠️ 無法復原（除非有備份）

## ❓ 常見問題

### Q1: 管理後台可以在哪裡使用？

**A**: 只能在本地電腦使用。不能直接在線上網站訪問（安全考量）。

### Q2: 資料儲存在哪裡？

**A**: 資料儲存在瀏覽器的 LocalStorage 中。建議定期匯出備份！

### Q3: 如果不小心刪除了資料怎麼辦？

**A**:
1. 如果有備份，使用「匯入資料」功能還原
2. 如果沒有備份，點擊「重置」回到預設資料
3. 最壞情況下需要重新輸入

### Q4: 為什麼編輯後網站沒有更新？

**A**: 編輯後需要：
1. 匯出資料
2. 執行 `npm run build`
3. 重新部署網站

### Q5: 可以多人同時編輯嗎？

**A**: 目前版本不支援。每個人的資料是獨立的。建議：
- 指定一個人負責編輯
- 或使用「匯出/匯入」功能同步資料

### Q6: 如何上傳圖片？

**A**: 目前版本需要手動處理：
1. 將圖片放到 `public/images/` 目錄
2. 在 JSON 中使用相對路徑 `/images/your-image.jpg`
3. 未來版本會新增圖片上傳功能

### Q7: 忘記備份就重置了怎麼辦？

**A**: 系統會自動保留一個備份。點擊頂部的「還原備份」按鈕試試看。

### Q8: 可以在手機上使用嗎？

**A**: 技術上可以（在手機瀏覽器開啟 `localhost:3000/admin`），但不建議，因為螢幕太小不好操作。

## 🎯 最佳實踐

### 1. 定期備份
- 每次編輯前先匯出一次
- 每週備份一次
- 重要修改前備份

### 2. 命名規範
- 工程 ID 使用英文，如 `taipei-building-2024`
- 產品 ID 使用英文，如 `mat-carpet-gray`
- 圖片命名有意義，如 `project-taipei-101-front.jpg`

### 3. 圖片管理
- 壓縮圖片後再上傳（建議 < 500KB）
- 使用一致的尺寸比例
- 使用 JPG 格式（照片）或 PNG 格式（透明圖）

### 4. 內容撰寫
- 描述清楚簡潔
- 標籤使用一致
- 定期檢查內容品質
