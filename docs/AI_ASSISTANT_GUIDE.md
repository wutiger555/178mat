## 什麼是 AI 助手？

AI 助手就像一位懂程式的助理，可以幫您：

✅ **撰寫**新案場的資料格式
✅ **檢查**您的資料有沒有錯誤
✅ **修改**現有案場的內容
✅ **生成**產品描述文字
✅ **回答**技術問題

## 推薦的 AI 工具

### 1. Claude（最推薦）

**優點：**
- 📝 擅長處理程式碼和資料格式
- 🔍 理解力強，可以看懂複雜的檔案
- 🛡️ 安全性高

**如何使用：**
1. 前往 https://claude.ai/
2. 註冊免費帳號
3. 開始對話

**費用：**
- 免費版：足夠日常使用
- Pro 版：$20/月（更快、更多次數）

### 2. ChatGPT（備選）

**優點：**
- 🌍 最普及的 AI 工具
- 📚 資料豐富

**如何使用：**
1. 前往 https://chat.openai.com/
2. 註冊帳號
3. 開始對話

**費用：**
- 免費版：GPT-3.5（夠用）
- Plus 版：$20/月（GPT-4，更聰明）

### 3. Cursor（進階）

**適合：** 已經熟悉 VS Code 的使用者

**特色：** 直接在編輯器中使用 AI

## 基本使用方式

### 步驟 1：準備資料

將現有的案場資料複製下來，讓 AI 了解格式。

**範例：**
```
我有一個網站專案，需要管理案場資料。
以下是一個案場資料的範例：

{
  id: "taichung-dali",
  title: "台中大里區住宅",
  location: "台中市大里區健民路3號",
  city: "台中",
  district: "大里區",
  year: 2025,
  images: ["/images/project-1.jpg"],
  description: "...",
  tags: { ... },
  specs: { ... }
}
```

### 步驟 2：清楚說明需求

告訴 AI 您想做什麼。

**❌ 不好的問法：**
```
幫我寫一個案場
```

**✅ 好的問法：**
```
請幫我新增一個案場資料，格式要跟上面的範例一樣：

案場名稱：高雄前鎮區辦公大樓
地址：高雄市前鎮區中山路100號
完工年份：2024
建築類型：辦公大樓
地面材質：地磚
安裝方式：嵌入式
框架：AL-620
面料：波羅紋毯面灰色
排水：導水溝
```

### 步驟 3：檢查回覆

AI 會生成完整的資料格式，您需要檢查：

✅ 格式是否正確（有沒有缺少逗號、括號）
✅ 資料是否符合您的需求
✅ `id` 是否唯一（沒有重複）

### 步驟 4：複製到檔案

將 AI 生成的內容複製到 `projects.ts` 檔案中。

### 案例 2：批量修改案場年份

**您的提問：**
```
我有 10 個案場的年份都是 2023，我想全部改成 2024。
請幫我寫一個查找和替換的方法。
```

**AI 的回覆：**
```
您可以使用 VS Code 的「尋找和取代」功能：

1. 按 Ctrl+H（Windows）或 Cmd+H（Mac）
2. 在「尋找」欄位輸入：year: 2023
3. 在「取代」欄位輸入：year: 2024
4. 點擊「全部取代」

或者，如果您想更精確，可以這樣搜尋：
尋找：year: 2023,
取代：year: 2024,

這樣可以避免誤改其他地方的 2023。
```

### 案例 3：檢查格式錯誤

**您的提問：**
```
我剛才新增了一個案場，但網站顯示錯誤。
請幫我檢查這段程式碼有什麼問題：

{
  id: "taipei-new-project"
  title: "台北新案場",
  location: "台北市信義區",
  city: "台北"
  district: "信義區",
  year: 2025,
}
```

**AI 的回覆：**
```
我發現了幾個格式錯誤：

1. 第 2 行：id 後面缺少逗號
   ❌ id: "taipei-new-project"
   ✅ id: "taipei-new-project",

2. 第 4 行：city 後面缺少逗號
   ❌ city: "台北"
   ✅ city: "台北",

3. 第 7 行：最後一項不應該有逗號
   ❌ year: 2025,
   ✅ year: 2025

修正後的版本：

{
  id: "taipei-new-project",
  title: "台北新案場",
  location: "台北市信義區",
  city: "台北",
  district: "信義區",
  year: 2025
}
```

```
請幫我新增一個案場資料，格式參考現有案場：

- 地址：[填入]
- 城市：[填入]
- 行政區：[填入]
- 完工年份：[填入]
- 建築類型：[填入]
- 地面材質：[填入]
- 安裝方式：[填入]
- 框架型號：[填入]
- 面料：[填入]
- 排水方式：[填入]

- 描述：[填入]

=== 檢查格式範本 ===

[貼上程式碼]

```

### 技巧 2：上傳整個檔案給 AI 分析

**步驟：**
1. 將 `projects.ts` 檔案複製
2. 在 Claude 中上傳（Claude Pro 支援檔案上傳）
3. 詢問：「請幫我分析這個檔案，有沒有格式錯誤或重複的 ID？」

### 技巧 3：請 AI 寫自動化腳本

**範例：**
```
我想寫一個腳本，自動檢查所有案場的圖片是否存在。
請幫我寫一個 Node.js 腳本。
```

AI 會提供完整的腳本程式碼和使用說明。

### 技巧 4：多輪對話優化內容

不要一次問完所有問題，可以逐步優化：

```
第 1 輪：請幫我生成新案場資料
第 2 輪：描述文字太短，請擴充到 100 字
第 3 輪：請加入更專業的用語
第 4 輪：請檢查格式是否正確
```

## 注意事項

### ⚠️ 安全性

1. **不要上傳敏感資料**
   - 避免包含客戶聯絡資訊
   - 避免包含價格資訊
   - 避免包含內部機密

2. **檢查 AI 的回覆**
   - AI 可能會出錯
   - 重要資料務必人工確認
   - 不要盲目複製貼上

3. **保留備份**
   - 修改前先備份檔案
   - 使用 Git 版本控制
   - 定期匯出資料

### 📝 最佳實踐

1. **明確的指令**
   - 說清楚您要什麼
   - 提供完整的資訊
   - 舉例說明

2. **逐步驗證**
   - 先測試一個案場
   - 確認無誤後再批量處理
   - 及時儲存進度

3. **保持對話連貫**
   - 在同一個對話中處理相關問題
   - AI 會記住前面的內容
   - 避免重複解釋

4. **學習累積**
   - 記錄有用的提示詞
   - 建立個人知識庫
   - 分享經驗給同事

## 常見問題

### Q1: AI 生成的格式一直有錯誤？

**A:**
1. 提供更完整的範例給 AI
2. 明確說明「請嚴格遵守格式」
3. 請 AI 先解釋格式規則，再生成內容

### Q2: AI 能否直接幫我修改檔案？

**A:**
- Claude Code 可以（需要特殊設定）
- 一般的 Claude/ChatGPT 只能提供程式碼，需要您手動複製
- 建議使用 Cursor 編輯器，可直接修改

### Q3: 每次都要重新解釋格式嗎？

**A:**
- 在同一個對話中不用
- 新對話需要重新提供範例
- 建議使用「Custom Instructions」功能儲存常用指令

### Q4: AI 會不會把我的資料外洩？

**A:**
- 正規 AI 服務（Claude、ChatGPT）有隱私保護
- 不要上傳機密資料
- 可以將敏感資訊替換成代號

### Q5: 我不懂程式，AI 能教我嗎？

**A:**
當然可以！例如：
```
我是完全不懂程式的新手，請用最簡單的方式教我如何新增案場。
請一步一步說明，像在教小學生一樣。
```
