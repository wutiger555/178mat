# 管理後台編輯功能完整教學

> 詳細說明如何使用新增和編輯功能

---

## 🎯 功能概覽

現在管理後台提供**完整的編輯表單功能**，不需要手動編輯 JSON 檔案！

✅ **工程實績管理** - 新增、編輯、刪除工程案例
✅ **產品管理** - 新增、編輯、刪除產品型錄
✅ **圖形化介面** - 類似 WordPress 的操作體驗
✅ **即時預覽** - 圖片和內容即時顯示
✅ **標籤管理** - 點擊選擇，簡單直覺

---

## 📁 工程實績管理

### 新增工程實績

#### 步驟 1：點擊新增按鈕

1. 進入「工程實績管理」頁面
2. 點擊右上角的「新增工程實績」按鈕
3. 會彈出編輯表單對話框

#### 步驟 2：填寫基本資訊

必填欄位標註 * 號：

- **工程名稱*** - 例：`台北市信義區住宅大樓`
- **城市*** - 例：`台北`
- **區域** - 例：`信義區`
- **年份** - 例：`2024`
- **完整地址*** - 例：`台北市信義區信義路五段...`
- **工程描述** - 詳細說明工程特色

💡 **提示**：
- 工程 ID 可以留空，系統會自動生成
- 如果要自訂 ID，建議使用英文和數字，例：`taipei-xinyi-2024`

#### 步驟 3：新增工程圖片

1. 在「工程圖片」區域，輸入圖片路徑
2. 格式：`/images/projects/project-name.jpg`
3. 點擊「新增」按鈕
4. 圖片會立即顯示預覽
5. 可以新增多張圖片
6. 如要刪除，滑鼠移到圖片上點擊「刪除」

📌 **圖片路徑範例**：
```
/images/projects/taipei-building-front.jpg
/images/projects/taipei-building-entrance.jpg
/images/hero-mat-1.jpg
```

#### 步驟 4：選擇標籤分類

點擊標籤按鈕即可選取/取消：

- **建築類型** - 住宅大廈、公共建築、學校等
- **地面材質** - 地磚、抿石子、石材等
- **安裝方式** - 嵌入式、表面式等

選中的標籤會變成彩色顯示。

#### 步驟 5：填寫規格資訊（選填）

- 面積 - 例：`50 平方公尺`
- 深度 - 例：`2.5 公分`
- 寬度 - 例：`120 公分`
- 長度 - 例：`300 公分`

#### 步驟 6：儲存

1. 確認所有資訊填寫完整
2. 點擊右下角「新增」按鈕
3. 看到成功提示訊息
4. 回到列表頁面，新工程已出現！

---

### 編輯工程實績

#### 步驟 1：找到要編輯的工程

1. 在工程列表中找到目標工程
2. 或使用搜尋框快速查找
3. 點擊工程卡片右側的「編輯」按鈕

#### 步驟 2：修改內容

1. 編輯表單會預先填入現有資料
2. 修改任何想要更新的欄位
3. 工程 ID 不可修改（灰色顯示）

#### 步驟 3：儲存變更

1. 點擊「儲存」按鈕
2. 看到「工程實績已更新」提示
3. 資料立即生效！

## 📦 產品管理

### 新增產品

#### 步驟 1：點擊新增按鈕

1. 進入「產品管理」頁面
2. 點擊右上角的「新增產品」按鈕

#### 步驟 2：填寫基本資訊

必填欄位：

- **產品名稱*** - 例：`波羅紋毯面 - 灰色`
- **產品分類*** - 例：`面料`
- **產品描述*** - 詳細說明產品特色
- **產品圖片路徑** - 例：`/images/products/carpet-gray.jpg`
- **價格**（選填）- 例：`NT$ 1,200 / 平方公尺`

💡 **提示**：輸入圖片路徑後會立即顯示預覽！

#### 步驟 3：填寫規格資訊

- **材質** - 例：`高密度聚丙烯纖維`
- **厚度** - 例：`約 12mm`
- **寬度** - 例：`可客製化`

#### 步驟 4：新增產品特點

1. 在「產品特點」輸入框輸入特點
2. 例：`優異除泥效果`
3. 點擊「新增」或按 Enter 鍵
4. 特點會變成標籤顯示
5. 可以新增多個特點
6. 點擊標籤上的 × 可刪除

📌 **常用特點範例**：
```
✓ 優異除泥效果
✓ 耐磨耐用
✓ 易於清潔
✓ 防滑設計
✓ 抗UV處理
```

#### 步驟 5：新增顏色選項

1. 輸入顏色名稱，例：`灰色`
2. 點擊「新增」或按 Enter
3. 可以新增多個顏色

#### 步驟 6：新增適用場所

1. 輸入場所類型，例：`住宅大廈`
2. 點擊「新增」或按 Enter
3. 常用場所：
   - 住宅大廈
   - 商業大樓
   - 公共建築
   - 學校
   - 醫院

#### 步驟 7：儲存

點擊「新增」按鈕，完成！

## 🖼️ 圖片管理技巧

### 圖片儲存位置

建議將圖片存放在：
```
public/images/projects/    # 工程圖片
public/images/products/    # 產品圖片
public/images/             # 其他圖片
```

### 圖片命名建議

使用有意義的檔名：
```
✅ 好的命名
taipei-building-entrance.jpg
product-carpet-gray-front.jpg
project-school-2024-1.jpg

❌ 避免這樣命名
IMG_1234.jpg
photo.jpg
1.jpg
```

### 圖片尺寸建議

- **工程圖片**：建議 1200x800px，橫式
- **產品圖片**：建議 800x600px
- **檔案大小**：壓縮後 < 500KB

## 💾 資料儲存機制

### 自動儲存

- 點擊「新增」或「儲存」後，資料會立即存到 LocalStorage
- 同時建立自動備份
- 關閉瀏覽器後資料仍然保留

### 手動備份

定期使用「匯出資料」功能：

1. 點擊頂部「匯出資料」按鈕
2. 下載 `cms-data.json` 檔案
3. 存到安全的地方（建議每週備份）

### 還原資料

如果不小心刪除：

1. 點擊「匯入資料」按鈕
2. 選擇之前匯出的 JSON 檔案
3. 資料立即還原

## ⚙️ 進階技巧

### 快速新增多個項目

1. 新增第一個項目
2. 填寫完整資訊後儲存
3. 立即點擊「新增」按鈕
4. 表單會重置，可以快速輸入下一個

### 複製類似項目

1. 找到相似的項目
2. 點擊「編輯」
3. 修改工程 ID 和必要欄位
4. 另存為新項目（目前需手動匯出匯入）

### 批次編輯

目前版本不支援批次編輯，建議：
1. 匯出 JSON 檔案
2. 使用文字編輯器批次修改
3. 匯入更新

## 🔍 常見問題

### Q1: 圖片不顯示怎麼辦？

**A**: 檢查以下幾點：
1. 圖片路徑是否正確？
2. 圖片檔案是否存在於 `public/` 目錄？
3. 路徑是否以 `/` 開頭？

正確格式：
```
✅ /images/project-1.jpg
❌ images/project-1.jpg
❌ ./images/project-1.jpg
```

### Q2: 如何修改已新增的圖片？

**A**:
1. 點擊「編輯」按鈕
2. 滑鼠移到圖片上
3. 點擊「刪除」移除舊圖片
4. 新增新的圖片路徑

### Q3: 標籤可以自訂嗎？

**A**:
- 目前標籤選項是預設的
- 如需新增標籤選項，需要修改 `src/data/cms-config.ts`
- 未來版本會支援自訂標籤

### Q4: 忘記備份就刪除了怎麼辦？

**A**:
- 系統有自動備份功能
- 點擊頂部的「還原備份」按鈕
- 如果沒有備份，只能重新輸入

### Q5: 可以上傳圖片嗎？

**A**:
- 目前版本需要手動將圖片放到 `public/images/` 目錄
- 然後在表單中輸入路徑
- 未來版本會新增拖曳上傳功能

## 🎓 學習路徑

建議按照以下順序學習：

1. **第一天** - 熟悉介面，瀏覽現有內容
2. **第二天** - 嘗試編輯一個工程實績
3. **第三天** - 新增一個測試工程
4. **第四天** - 管理產品，編輯和新增
5. **第五天** - 完整流程：新增→編輯→備份→發布

---

## 💡 最佳實踐

1. **編輯前備份** - 養成習慣，每次編輯前先匯出
2. **填寫完整** - 盡量填寫所有欄位，提供完整資訊
3. **使用標籤** - 善用標籤分類，方便搜尋和篩選
4. **命名規範** - ID 使用英文，保持一致性
5. **圖片最佳化** - 壓縮圖片，提升載入速度
6. **定期整理** - 刪除不需要的舊資料
7. **測試後發布** - 編輯完成後先檢查再 build
